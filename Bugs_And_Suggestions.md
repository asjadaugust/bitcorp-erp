# Bugs and Suggestions

This document summarizes the findings from the Product Owner testing session for workstreams WS-24, WS-25, WS-26, WS-29, and WS-30.

## 🐞 Bugs Found

### WS-25: Analytics Dashboard - Fleet Utilization API Error

- **Issue**: The API endpoint `/api/analytics/fleet/utilization` returns a `500 Internal Server Error`.
- **Backend Error**: `TypeError: Cannot read properties of undefined (reading 'map')` in the context `Analytics.fleetUtilization`.
- **Impact**: The "Resumen de Utilización de Flota" (Fleet Utilization Summary) and corresponding charts cannot be displayed, leaving the analytics dashboard partially broken.
- **Probable Cause**: A variable expected to be an array in `EquipmentAnalyticsService.getFleetUtilization` or the corresponding DTO transformer is `undefined` when attempting to call `.map()`.

### WS-30: Operator List - Empty due to Tenant ID Mismatch (Critical)

- **Issue**: The `/api/operators` list returns "No hay datos disponibles" for the default `admin` user.
- **Root Cause**:
  1. The `OperatorService.findAll` method filters by `tenantId` using `t.tenantId = :tenantId`.
  2. The `admin` user has no `tenant_id` assigned (seeded as NULL).
  3. The `JwtPayload` assigns `id_empresa: 0` when `unidad_operativa_id` is missing.
  4. SQL `tenant_id = 0` fails to match records where `tenant_id` is `NULL`.
- **Impact**: Default administrative users cannot see the seeded operators.
- **Fix**: Update the query to handle `NULL` tenant IDs or ensure seeders assign a default `tenant_id` (e.g., 1) to both users and trabajadores.

### WS-30: Operator Profile - Data Wiring

- **Issue**: Some operator fields in the profile view (e.g., specific skills or performance history) appear empty or use stubbed values.
- **Impact**: Incomplete user profile experience for operators.
- **Suggestion**: Ensure all backend methods in `OperatorService` are fully implemented and the frontend is correctly binding to the returned data.

## 💡 Suggestions for Improvement

### General UX: Aero Design System Consistency

- **Dropdowns**: Standardize the use of `app-dropdown` across all forms (Payments, Operators, Contracts) to replace native `<select>` tags for a premium feel.
- **Loading States**: Add skeleton loaders or more prominent loading indicators for analytics charts and data-heavy lists to improve perceived performance.

### WS-24: Notifications Panel

- **"Mark All as Read"**: Ensure this action provides immediate UI feedback (e.g., badge clearing instantly) without requiring a full page refresh or long polling delay.
- **Notification Persistence**: Check if notifications persist after logout/login and if the "Leído" status is correctly synced across devices.

### WS-29: Payment Excel Export

- **Export Filters**: Allow users to export only the filtered results currently visible on the screen, rather than the entire dataset, to provide more flexibility.
- **Feedback**: Show a "Generando archivo..." indicator when clicking the "Exportar" button to let the user know the process has started.

### WS-26: Operator Availability/Performance

- **Visual Indicators**: Use color-coded status badges for availability (e.g., Green for Available, Red for Not Available) and progress bars for performance metrics on the operator list page for quick scanning.
